# 飞书 Markdown 消息卡片设计

## 目标

将当前飞书机器人发送的纯文本周报改为可渲染标题、加粗和列表的
`interactive` 消息卡片，并保证长日报不会因单条消息过长而失败。

## 设计

- 保留 Markdown 文件作为归档源，不修改报告生成提示词。
- 按 `##` 二级标题拆分日报；每个章节生成一张飞书卡片。
- 卡片正文使用 `lark_md`。
- Markdown 表格转换为逐行加粗标签，避免飞书不支持表格时显示原始竖线。
- 超长章节按空行和字符上限继续拆分。
- 短快照也走相同卡片发送函数，通常只生成一张卡片。
- 每张卡片独立签名并检查飞书返回码；任一失败即让任务失败并留日志。

## 验收

- 发送请求的 `msg_type` 为 `interactive`。
- 卡片正文节点的文本类型为 `lark_md`。
- 长日报拆成多张卡片。
- 表格不再包含 Markdown 分隔行。
- 本期日报重新推送后，飞书返回成功。
